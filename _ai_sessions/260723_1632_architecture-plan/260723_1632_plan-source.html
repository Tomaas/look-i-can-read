Problem — media-store claims the /-prefix vs https:// rule as its back-compat boundary, but resolveReferenceImage re-implements the whole branch (own BLOB_HOST_SUFFIX, own allowlist compare), and edge TTS — the default provider — bypasses saveMedia entirely, so audio silently breaks in Blob mode and the two TTS adapters don't honor the same contract.

Solution — media-store exports the read-back (resolveStoredMediaForModel) hiding both branches; edge TTS synthesizes to bytes and saves through saveMedia like elevenlabs.

  • storage rule stops leaking across the seam
  • TTS adapters become truly substitutable
  • Blob deploys get working audio
  • leverage: one read-back, N consumers
  • goldens can pin both branches at one interface
